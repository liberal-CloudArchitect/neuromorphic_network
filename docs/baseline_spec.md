# 公平基线规格（DRAFT）

> 当前只锁定“需要比较什么”的原则，具体架构、容差和预算仍待 P0 评审。

## 比较层级

| 层级 | 目的 | 必须匹配/记录 | 结论用途 |
|---|---|---|---|
| 参数匹配单体 RNN/Transformer | 主比较 | 可训练参数量、输入输出接口、任务数据、调参预算 | 判断模块化收益是否超出容量增加 |
| 训练计算匹配单体模型 | 敏感性 | token/环境步数、优化器步数、调参次数、wall-clock | 判断收益是否来自更多训练计算 |
| 推理成本匹配单体模型 | 敏感性 | 活跃 FLOPs、端到端延迟、batch/device/precision | 判断收益是否在相近部署成本下成立 |

无法同时匹配的维度必须量化披露，禁止选择性只报告有利基线。

## 必备弱基线与结构对照

- 随机策略/多数类与任务最小启发式；
- 简单记忆与无记忆模型；
- 固定路由、随机路由和稠密路由；
- 无工作区、无预测器等逐项消融；
- 冻结随机/浅层容量匹配编码器；
- 直接 policy head 与候选冲突扰动。

## 公平性检查

- [ ] 数据、划分、seed、评价实现和早停规则相同。
- [ ] 每类模型获得预注册且可比较的调参预算。
- [ ] 主结果以参数匹配模型为比较对象；其余两类标为敏感性分析。
- [ ] 失败 run、资源超限和数值异常均计入审计，不做事后选择。
- [ ] 参数、计算、延迟和存储成本随结果一并发布。
