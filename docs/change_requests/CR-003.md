# CR-003：以跨步预测闭环和 semantic top-1 路由启动 P4

## 变更理由

P3 的正式实验保持有效，但 `predictive_adapter.v1` 未获得预注册因果支持，稀疏执行也超过任务分数非劣界。为检验修订机制而不改写 P3，本变更建立独立的 `p4-protocol-v1`、P4 数据 namespace 和 `GATE-NN-MVP-v2`。

## 批准变更

- 新增 `predictive_adapter.v2`：只消费前一步动作后提交的 forecast，当前步形成有界误差反馈，当前动作形成后再提交下一步 forecast。
- 新增 `sparse_router.v2`：optional experts 仅为 episodic / working，每个有效 token semantic top-1；AR STORE/QUERY 100% reservation 到 episodic，且不允许 capacity drop。
- 冻结 P4 split seeds：train `11101`、validation `12201`、analysis `15501`、test `13301`、OOD `14401`。
- 冻结 seed `7` 的 8-cell qualification 和 4-cell pilot、三正式 seeds 的 24-cell mechanism，以及包含该机制矩阵的 81-cell full。
- 冻结 chance、ID、OOD、AULC、forgetting、预测因果、预测质量和稀疏执行定义与门槛，详见 `docs/p4_implementation_spec.md`。
- 冻结三个 seed、10,000 次配对 bootstrap、RNG `20260715`、95% CI 和 Holm-adjusted `p <= 0.05`。

## 关键门槛

ID 相对两个主基线均 ≥`5%`；SmallGraph live-rollout OOD 相对两个主基线均 ≥`5%`；AULC 相对两个主基线均 ≥`15%`；`baseline forgetting - modular-v2 forgetting` 对两个基线均 ≥`0.02`。预测因果要求 full 相对 retrained predictor-off 的 macro AULC ≥`5%` 且每任务最终下降 ≤`2pp`；预测质量要求 forecast error 相对 persistence 至少降低 `5%` 且至少两个任务方向为正。稀疏执行要求 optional MAC 降低 ≥`20%`、每任务损失 ≤`2pp`、零 drop，以及 AR STORE/QUERY episodic 100% reservation/execution。

## 兼容与历史

- P4 目标软件版本为 `0.5.0`；v2 module ID 不替换或重命名 P3 的 v1 ID。
- P3 的数据、artifact、`p3-protocol-v2`、`GATE-NN-MVP FAILED` 和 `0.4.0` 历史不得回写。
- qualification、pilot、mechanism 和 full 必须依次由 clean SHA 的 lock 衔接；阶段通过不得替代 `GATE-NN-MVP-v2` 科学裁决。
- 改变任何 split、seed、预算、矩阵、preset、指标、阈值、统计 family 或科学边界时，必须发布新 protocol version 和新 change request，不得修改本记录。
