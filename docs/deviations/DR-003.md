# Formal summary 缺少 capacity-drop 计数

正式三 seed summary 记录了 optional expert 的真实调用、active/dense MAC 和 profiler
coverage，但没有保存 capacity-drop 计数。因此可以确认三个专家均被调用、没有调用塌缩，
却不能仅从 formal artifact 独立证明 `capacity drops = 0`。

P2 与 qualification 对容量行为已有工程测试，但 `GATE-NN-MVP` 要求正式证据，本次仍按
缺失处理。不得用早期 fixture 替代正式证据来反转科学 Gate；后续 protocol 应把容量统计
加入 cell summary 与 checksum 清单。
