# 六模块计算假说（DRAFT）

> 下列内容是待评审的计算规格，不冻结 `BrainPacket`、`ModuleOutput`、张量形状或类接口，也不表示 `GATE-0` 已通过。

| 模块 | 最小职责 | 状态/时间尺度 | 候选损失 | 预注册对照 | 可测预测与否证条件 |
|---|---|---|---|---|---|
| 感觉编码 | 将 observation 转为任务可共享的潜在表示 | 当前输入；可选短时上下文 | 任务损失、重建/对比辅助损失 | 冻结随机编码器、浅层容量匹配编码器 | OOD/probe 改善；替换后无恶化则功能主张不成立 |
| 情景记忆 | 快速写入事件、模式分离、线索检索 | episode 内快速状态；跨 batch 隔离 | 检索、分离、重建损失 | 无记忆、简单记忆、禁写/禁读 | 相对无记忆提升 ≥15pp 且 95% CI 不跨 0，否则不通过 |
| 工作记忆 | 保持规则/目标并控制何时更新 | 容量受限、跨延迟步保持 | 任务损失、状态一致性/门控正则 | 无工作区、状态清零、门控扰动 | 消融使延迟/规则任务下降 ≥10pp 且 95% CI 不跨 0 |
| 预测适配 | 接收 `s_t` 与动作副本 `a_t`，预测 `s_(t+1)` | 转移级；误差在环境反馈后计算 | 下一状态、轨迹或不确定性损失 | 无预测器、乱序动作副本、容量匹配辅助头 | 消融使 probe 或样本效率恶化 ≥10%，否则主张失败 |
| 动作选择 | 聚合候选、目标、预测与冲突为策略输出 | 决策步级；可选冲突状态 | policy/value、冲突与校准损失 | 直接 policy head、候选置乱/冲突扰动 | 冲突条件下产生可重复收益；直接 head 同等则缩减模块 |
| 稀疏路由 | 在任务特定 `eligible_experts` 内选择计算与通信路径 | token/step 级路由状态 | 任务损失、负载平衡、通信成本 | 固定路由、随机路由、稠密路由 | eligible set ≥4 时平均激活率 ≤60%；塌缩或无 FLOPs 收益则失败 |

## 共同约束

- 必经模块不进入路由稀疏率分母，路由负载按预期任务分布审计。
- 模块必须可替换、可消融、可单测，并显式声明状态初始化、更新、分离和清理时机。
- 主任务损失与辅助损失分别记录；梯度冲突和权重敏感性必须报告。
- 模块可显式产出有界、已归约 telemetry，但 telemetry 不是模型决策输入。
- 任何生物功能类比都必须链接到 `evidence_registry.md` 的证据、抽象与否证条件。
