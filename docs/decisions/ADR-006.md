# ADR-006：预测闭环必须跨步，记忆稀疏性必须保持语义

## 背景

P3 已在 `p3-protocol-v2` 下完成并得到 `GATE-NN-MVP FAILED`。其中 `predictive_adapter.v1` 的训练消融没有达到因果门槛，旧稀疏路径虽节省 MAC，却超过任务分数非劣界。P4 需要通过新版本机制回答这两个失败点，同时保持 P3 证据不可变。

## 决策

采用 `p4-protocol-v1`：

- `predictive_adapter.v2` 在第 `t` 步消费第 `t-1` 步动作后提交的 forecast，以当前 sensory representation 计算误差和不超过 `0.25` 的反馈；第 `t` 步动作形成后才提交下一 forecast。episode 边界清除状态，禁止同一步或未来信息泄漏。
- `sparse_router.v2` 只在 `episodic_memory.v1` 与 `working_memory.v1` 间执行 semantic top-1。AR STORE/QUERY 强制 episodic reservation；其余 token 使用 learned stable top-1；所有有效 token 精确执行一个 optional expert，drop 必须为零。
- 使用独立 P4 split seeds，并把 qualification、pilot、24-cell mechanism 与 81-cell full 作为有序、带 hash 的冻结阶段。
- 最终收益由 chance-normalized ID、SmallGraph live-rollout OOD、AULC、绝对 forgetting reduction `0.02`、预测因果、预测质量和稀疏非劣性共同约束，交由 `GATE-NN-MVP-v2` 裁决。

## 选择理由

跨步 consume/commit 顺序使 forecast 真正参与下一步决策，同时保留可审计的时间因果边界。把 episodic / working 限定为 top-1 optional experts，使 MAC 节省来自真实未执行，而 AR reservation 避免学习路由破坏已知必要语义。单独的 mechanism gate 可在投入 81-cell full 前否决无效闭环。

## 后果

- mechanism 固定为每 seed 8 个 cell，三个正式 seed 共 24 个；full 固定为 81 个 cell。
- pilot 只可依据 train/validation 上的预测资格和 macro AULC 选择 preset；test/OOD 不得进入选择。
- 正式统计使用三个 seed、10,000 次配对 bootstrap、RNG `20260715`、95% CI 和 Holm 校正。
- 目标软件版本为 `0.5.0`，但软件版本号本身不表示 `GATE-NN-MVP-v2` 通过。

## 历史与版本约束

`predictive_adapter.v2`、`sparse_router.v2` 和 P4 namespace 与 P3 v1 路径并存。不得迁移、覆盖或重算 P3 artifact；P3 Gate 状态保持 FAILED。任何对架构顺序、split、seed、矩阵、预算、preset、指标、阈值、统计 family 或 Gate 语义的变更都必须建立新 protocol 和 change request，不得回填 `p4-protocol-v1`。
